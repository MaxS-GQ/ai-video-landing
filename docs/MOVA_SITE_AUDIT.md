# MOVA Site — Аудит существующего проекта

Дата: 2026-07-13. Репозиторий: `MaxS-GQ/ai-video-landing`.
Продакшн: https://maxs-gq.github.io/ai-video-landing/ (GitHub Pages, legacy build из `main`).

## 1. Текущая архитектура

| Аспект | Состояние |
|---|---|
| Стек | Чистый HTML + inline CSS + inline vanilla JS, **один файл** `index.html` (~34 КБ, ~840 строк) |
| Сборка | Отсутствует. Нет package.json, зависимостей, lint, TypeScript |
| Страницы | Одна (`/`). Навигация — якоря внутри страницы |
| Компоненты | Нет. Всё — разметка в одном файле |
| Стили | CSS-переменные в `:root` (частичные токены), остальное — ad-hoc значения |
| Анимации | CSS keyframes (orbs, звёзды, marquee, телефон, демо на emoji) + 2 маленьких JS-скрипта (IntersectionObserver reveal, cursor glow, tilt) |
| Медиа | 3 PNG из одного логотипа (`mova-logo.png` 472 КБ с запечённым фоном, favicon, touch-icon). Фото/видео продукта нет |
| Шрифты | Системный стек (SF Pro / -apple-system). Веб-шрифты не подключены |
| Деплой | GitHub Pages legacy (ветка `main`, корень). Работает, пересборка ~30 сек |
| Локальный запуск | `python3 -m http.server` (конфиг `mova-site` в launch.json проекта приложения). Проверено — работает |

## 2. Сильные стороны (сохранить)

- **Мгновенная загрузка**: ноль зависимостей, один запрос HTML + 3 картинки.
- **Тёмная идентичность и брендовый синий** — совпадает с приложением MOVA; направление палитры верное.
- **Логотип** (кольцо «MO») — узнаваемый актив; исходник 1254² у пользователя в `~/Downloads/MOVA-logo.png`.
- **Английские тексты** — рабочая основа копирайтинга (features, how it works, trust, контакты).
- **Рабочий бесплатный деплой** GitHub Pages + чистая git-история с понятными коммитами.
- Учёт `prefers-reduced-motion` уже есть.

## 3. Визуальные проблемы (по новым правилам дизайна — см. MOVA_DESIGN.md)

- **Запрещённые клише присутствуют**: сине-фиолетовый AI-градиент в заголовке; звёздное небо + sparkles ✨; случайные светящиеся орбы; neon glow; бегущая строка (marquee); россыпь badge-капсул.
- **Emoji как основная графика** (🕺👩👨👶🎬) — выглядит детски, противоречит «премиальному» характеру.
- **Одинаковые стеклянные карточки** для каждого блока — features, steps, trust, contact визуально не различаются.
- Glass-эффекты — «полупрозрачные прямоугольники с blur», без имитации материала/преломления/направленного света.
- Логотип-картинка с запечённым фоном требует маски/blend — хрупкое решение (артефакты уже ловили).

## 4. UX-проблемы

- **Один длинный скролл** — нет отдельных страниц функций, showcase, pricing, about, download.
- Нет реального интерактива: пользователь ничего не переключает и не пробует; демо — зацикленные emoji-анимации.
- Нет переключения сценариев (Future Baby / Trends / Motion Control) — суть продукта не проживается.
- Нет тарифов и кредитной системы вообще.
- Marquee и cursor-glow — декорация без функции.
- Мобильное меню отсутствует (nav-ссылки просто скрыты на <900px).

## 5. Технические проблемы

- Монолит: контент, стили, поведение и данные в одном файле — не масштабируется на 8+ страниц.
- Нет типизации, lint, сборки — регрессии ловятся только глазами.
- Данные (тексты, будущие цены) не отделены от разметки.
- SEO: нет canonical, sitemap, robots, structured data; OG-image — относительный путь (не работает в соцсетях).
- Иконка/лого: один непрозрачный PNG на все случаи; нет WebP/AVIF, нет srcset.

## 6. Производительность

- Сейчас быстро (нечему тормозить), но: 472 КБ PNG логотипа; множественные `backdrop-filter` + бесконечные CSS-анимации (orbs blur 90px, звёзды, marquee) заметно грузят GPU на слабых устройствах; `backdrop-filter` внутри анимированного контейнера уже давал артефакты рендера (чинили).
- Нет стратегии lazy-loading медиа (пока и нечего лениво грузить).

## 7. Что сохранить / что переработать

**Сохранить:** хостинг и репозиторий; логотип (пересобрать в прозрачный/маскированный ассет); направление палитры (глубокий графит-нави + брендовый синий); основу текстов; понятные коммиты.

**Переработать полностью:** архитектуру (миграция на Astro — см. план), визуальный слой по MOVA_DESIGN.md (убрать все клише из §3), демо-графику (emoji → продуманные CSS/SVG-сцены, затем реальные медиа по MOVA_MEDIA_REQUIREMENTS.md), навигацию (полноценный header + мобильное меню), SEO-слой.

## 8. Решение по стеку

Текущий «стек» (один HTML-файл) объективно мешает требованиям: 8+ страниц, компоненты, данные отдельно, TS/lint/build, SEO. Это не «замена ради вкуса», а необходимость.

**Выбор: Astro 4** (не 5 — на машине Node v20.2.0, Astro 5 требует ≥20.3):
- статический вывод → остаёмся на GitHub Pages бесплатно;
- нулевой JS по умолчанию, интерактив — точечными скриптами (совпадает с целью «минимальный JavaScript»);
- компоненты + scoped styles + TypeScript из коробки;
- деплой — официальный GitHub Actions workflow (переключить Pages с legacy на workflow).

React/Next не берём: серверные функции не нужны, React-runtime — лишние килобайты для сайта-витрины. GSAP/Three.js — не добавлять, пока конкретная сцена не потребует (первый срез делается на CSS + ванильном TS).

## 9. Рекомендуемая очередность

1. **Фундамент**: скаффолд Astro, дизайн-токены, BaseLayout (SEO/OG/JSON-LD), данные в `src/data/`.
2. **Вертикальный срез**: SiteHeader + мобильное меню + интерактивный hero с переключением трёх режимов + footer. Довести до эталонного качества.
3. **Страницы функций**: `/features` (переключатель) → `/features/future-baby` → `/features/trends` → `/features/motion-control`.
4. **Showcase и Pricing** (цены централизованы в `src/data/pricing.ts` с шага 1).
5. **About, Download**, юридические заглушки (Privacy/Terms — после готовности текстов).
6. **Оптимизация**: Lighthouse, Safari, mobile, a11y-проход, чистка.

Откат любого этапа — через git (каждый этап = отдельный коммит; старый сайт остаётся в истории коммитом `b9ea5a4`).
