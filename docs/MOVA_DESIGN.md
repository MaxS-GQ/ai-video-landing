# MOVA Design System

Главный источник правил для всех изменений сайта. Любой новый экран, компонент или анимация сверяется с этим документом. Токены из §4 — единственный источник значений: новые «магические числа» в компонентах не появляются.

---

## 1. Визуальный характер

Сайт воспринимается как **премиальный цифровой продукт**: современная визуальная AI-платформа, эмоциональная, но не детская; технологичная, но не холодная; футуристичная, но не «типичный AI-стартап»; минималистичная, но не пустая; кинематографичная, но удобная.

Формула образа: **тёмная студия с направленным светом**. Не «космос со звёздами», а съёмочный павильон: глубокий нейтральный фон, один ключевой источник света, материалы, которые этот свет преломляют. Каждая фича MOVA меняет **температуру света** сцены — так продукт «переодевает» сайт, как он переодевает пользователя.

## 2. Основной стиль — Liquid Glass

Стекло — это **материал**, а не полупрозрачный прямоугольник с blur. Рецепт материала (реализован токенами `--glass-*`):

1. **Заливка** — белый с низкой альфой (4–10%), чуть холоднее фона.
2. **Размытие подложки** — `backdrop-filter: blur() saturate(140%)`; blur растёт с «высотой» слоя.
3. **Кромка** — 1px внешняя граница (white 10–18%) **плюс** внутренний верхний блик `inset 0 1px 0` — свет ложится на верхнюю грань.
4. **Направленный свет** — глобальный key light **сверху-справа**: все sheen-градиенты под одним углом (150deg), блики смещены к правому верхнему углу. Никогда не смешивать направления света в одной сцене.
5. **Глубина** — мягкая ambient-тень вниз + затемнение нижней внутренней кромки. Чем выше слой, тем сильнее blur, светлее заливка и глубже тень.
6. **Преломление** — на крупных стеклянных объектах допустим тонкий внутренний градиент-каустик (едва заметное искривление света у кромок). Не имитировать дёшево — лучше отказаться, чем переборщить.

Три уровня материала: `glass-1` (спокойные поверхности, карточки в покое) → `glass-2` (приподнятые: header при скролле, активные панели) → `glass-3` (оверлеи: мобильное меню, модалки).

**Ограничение производительности:** не более двух уровней `backdrop-filter` друг над другом в одной точке экрана; внутри transform-анимируемых контейнеров backdrop-filter не использовать (артефакты рендера в WebKit/Chromium — проверено на прошлой версии).

## 3. Запрещено

- Шаблонный сине-фиолетовый AI-градиент (в т.ч. в тексте заголовков).
- Звёздочки, sparkles, «звёздное небо».
- Случайные светящиеся сферы/орбы как фон.
- Чрезмерный neon glow; свечение допустимо только как **мотивированный** свет от конкретного объекта (логотип, активный элемент) и в малых дозах.
- Одинаковые карточки для каждого блока; каждый смысловой блок получает собственную композицию.
- Стандартная SaaS-вёрстка «hero → 3 карточки → 3 шага → CTA» без вариаций.
- Россыпь badge/chip-элементов.
- Бегущие строки.
- Стены текста; длинные объяснения — ниже интерактивной части, коротко.
- Анимации ради анимаций (каждая объясняет переход или состояние).
- Копирование чужих сайтов/компонентов один в один; emoji как основная графика.

### Движение
| Токен | Значение | Роль |
|---|---|---|
| `--dur-micro` | `120ms` | hover, нажатия |
| `--dur-ui` | `240ms` | раскрытия, меню |
| `--dur-scene` | `480ms` | смена панелей, режимов |
| `--dur-hero` | `800ms` | трансформация hero-сцены |
| `--ease-standard` | `cubic-bezier(0.2, 0, 0, 1)` | По умолчанию |

## 5. Принципы анимации

- Анимация = объяснение перехода. Смена режима hero — это смена света и содержимого одной сцены, а не fade всей страницы.
- CSS — для состояний; ванильный TS — для оркестрации классов/атрибутов. GSAP/R3F подключаются только когда конкретная сцена этого требует (решение фиксируется в PLAN).
- Не использовать одинаковый fade-up для всех элементов; допустимые паттерны: смена глубины (scale 0.98→1 + blur→фокус), сдвиг световой температуры, каскад панелей с задержками 40–80ms, drag/before-after.
- Анимируются только `transform`, `opacity`, `filter`; никаких layout-свойств.
- Бесконечные анимации — только там, где есть «живой» объект (свет логотипа), и обязаны быть дешёвыми (opacity/transform).

## 6. Доступность

Семантика (`header/nav/main/section/footer`, один `h1` на страницу); полная клавиатурная навигация; видимый фокус (`outline: 2px solid var(--brand)`, offset 3px — единый стиль); переключатели режимов — паттерн tablist/radiogroup со стрелками; контраст текста ≥ 4.5:1 (`--text-mid` на `--bg-0` проходит); `alt` для значимых изображений, `aria-hidden` для декоративных; интерактив недоступный только через hover — запрещён; reduced-motion — см. §4.
