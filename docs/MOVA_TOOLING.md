# MOVA Site — Инструменты и подключения

Дата: 2026-07-13. Ветка работ: `site-modernization` (создана от `main@d77d3c6`).

## 1. Репозиторий и архитектура

- **Репозиторий:** `MaxS-GQ/ai-video-landing` (существующий, новый не создавался; git-история сохранена с первого коммита `b852fc8`).
- **Стек:** Astro 4.5.18 (static output) + TypeScript strict, ванильные скрипты для интерактива. Package manager — npm (lock-файл в репо).
- **Маршрутизация:** файловая, `src/pages/` (`/`, `/download`; план остальных — `docs/MOVA_IMPLEMENTATION_PLAN.md`).
- **Дизайн:** токены `src/styles/tokens.css` по `docs/MOVA_DESIGN.md`; компоненты в `src/components/`; данные в `src/data/`.
- **Deployment:** GitHub Pages, build_type=workflow (`.github/workflows/deploy.yml`, `withastro/action@v3`, Node 20). Деплой — только при push в `main`; рабочая ветка не деплоится.
- **Локальный запуск:** `npm run dev -- --port 4310` → `http://localhost:4310/ai-video-landing/` (конфиг `mova-site` в launch.json Claude Code).
- **Особенность окружения:** локальный Node 20.2.0 (Astro 5 требует ≥20.3 — потому Astro 4); npm-кэш пользователя повреждён root-owned файлами → локальный `.npmrc` (в `.gitignore`) переносит кэш в `~/.npm-mova-cache`.

| Лицензия | MIT |

| Обновление | `npx -y ui-ux-pro-max-cli@latest init --ai claude --global` (повторный запуск перезаписывает) или `uipro update` при глобальной установке CLI |
| Отключение | Удалить каталог `~/.claude/skills/ui-ux-pro-max/` (или `npx ui-ux-pro-max-cli uninstall --global`). Проект не затрагивается: в репо файлов скилла нет |

**Что используется:** базы стилей (67), палитр (161), шрифтовых пар (57), UX-гайдлайнов (99), правил по типам продуктов (161) через поисковый движок:

```bash
python3 ~/.claude/skills/ui-ux-pro-max/scripts/search.py "<запрос>" -d style|color|ux|typography|landing|product --max-results N
```

**Статус видимости в Claude Code:** файлы установлены корректно; реестр скиллов сессии формируется при старте, поэтому в текущей сессии Skill-tool его ещё не видит (проверено вызовом — «Unknown skill»; README скилла сам требует перезапуск ассистента). До перезапуска движок используется напрямую через `scripts/search.py` — функционально это те же данные. Проверка боевым запросом выполнена: домен style вернул релевантные результаты.

**Роль и приоритет:** экспертная справочная база (UX-правила, иерархия, типографика, a11y-чеклисты), НЕ генератор внешнего вида MOVA. При конфликте: MOVA_DESIGN.md → удобство пользователя → производительность → accessibility → рекомендации скилла. Примечательно: рекомендация базы для тёмных премиальных продуктов (hairline-бордеры `rgba(255,255,255,0.08)`, easing `cubic-bezier(0.16,1,0.3,1)`, запрет чистого `#000`, «ambient light») независимо совпала с уже принятыми токенами MOVA — используем её как валидатор, а не источник.

## 3. Magic MCP / 21st.dev — ПРОВЕРЕН, ПОКА НЕ ПОДКЛЮЧЁН

| | |
|---|---|
| Источник | https://github.com/21st-dev/magic-mcp |
| Подключение | `npx @21st-dev/cli@latest install claude --api-key <KEY>`; ключ — из https://21st.dev/magic/console (нужен аккаунт владельца проекта) |
| Модель | Free-tier мал (единицы генераций), дальше платно |
| Что делает | Генерирует **React + Tailwind/shadcn-ui** компоненты по описанию (`/ui ...`) |

**Решение:** в реестре MCP Anthropic коннектора нет; подключение возможно только с API-ключом владельца. Главное — **несовместимость вывода со стеком**: сайт — Astro-компоненты со scoped CSS на токенах MOVA, без React-runtime и без Tailwind. Принятие сгенерированного кода означало бы добавление React+Tailwind ради отдельных компонентов — прямое нарушение правил брифа (несовместимые зависимости, целая UI-библиотека ради одного компонента, конфликт стилевых систем). Поэтому Magic держим как **справочник паттернов** (структура диалогов, табов, слайдеров подсматривается на 21st.dev в браузере), а техническую основу интерактивных компонентов пишем нативно: ARIA Authoring Practices + UX-база скилла. Если появится задача, где Magic оправдан, — запросить у владельца API-ключ и подключить командой выше; удаление — убрать блок `@21st-dev/magic` из MCP-конфига Claude Code.

## 4. Refero — РУЧНОЙ РЕЖИМ

Refero (https://refero.design) — платная галерея референсов интерфейсов, без публичного API и MCP. Использование: анализ референсов в браузере (аккаунт владельца; для Claude — через браузерное расширение при необходимости). Результат анализа фиксируется ТОЛЬКО как переработанные принципы в `docs/MOVA_DESIGN.md` (раздел «Происхождение решений»): принцип → почему подходит MOVA → как переработан → отличие от источника. Прямое копирование макетов запрещено дизайн-системой (§3).

## 5. Прочие подключённые инструменты сессии

- **Preview-сервер Claude Code** — конфиг `mova-site` (astro dev, порт 4310) для скриншотов/инспекции.
- **GitHub CLI (`gh`)** — управление Pages/workflow.
- У пользователя в `~/.claude/skills/` есть и другие дизайн-скиллы (`design-system`, `ui-styling`, `brand`, `banner-design`, `slides`) — к проекту MOVA не привязывались, конфликтов нет (скиллы активируются по запросу).

## 6. Риски

1. **Astro 4 (не 5) из-за Node 20.2** — при обновлении Node до ≥20.3/22 стоит мигрировать на Astro 5 (низкий риск, статический сайт).
2. **UI UX Pro Max** — риск шаблонизации: база предлагает «универсальные» стили; купируется приоритетом MOVA_DESIGN.md и запретом §3 (клише). В репо не вносится, на bundle не влияет (нулевое влияние на прод).
3. **Magic MCP** — при бездумном использовании затащит React+Tailwind и сломает целостность стилей; поэтому статус «не подключён», использование только по явному решению.
4. **Refero** — риск прямого копирования чужих макетов; правило «принципы, не пиксели» зафиксировано в §4 и MOVA_DESIGN.md.
5. **Платные лимиты** Magic free-tier — не полагаться в пайплайне.
